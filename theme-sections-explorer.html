Safety Guardrails

Layer 1 — Prompt Constraint

Claude is told: "Edit ONLY between @theme: markers. If you need to change anything outside a marker, stop and explain why." This prevents accidental layout or logic changes.

Layer 2 — Post-Edit Validation

After Claude finishes, the server diffs all content outside theme markers. If anything changed, the edit is rejected, app.jsx is restored from backup, and the user is notified.

Backwards Compatibility

Apps without @theme: markers fall back to the current full-file restyle. No existing apps break.
